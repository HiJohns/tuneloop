tuneloop_dev client 调用 RegisterCustomerPermissions 返回 403 insufficient permissions

## 问题描述
TuneLoop 后端（tuneloop_dev client）在启动时尝试调用 IAM 的 `PUT /api/v1/namespaces/:id/customer-permissions` 注册业务权限（cus_perm），但收到 403 错误：

```
[PermissionRegistry] Failed to register permissions with IAM: 
RegisterCustomerPermissions returned status 403: {"error":"insufficient permissions"}
```

## 影响
- TuneLoop 无法注册 cus_perm 权限定义到 IAM
- 后续 `SetUserCustomerPermissions` 调用可能也受影响
- 批量导入用户时无法设置用户的 cus_perm，导致所有用户的 `cus_perm = 0`
- 前端权限检查失效（如 site_admin 无法看到库存菜单）

## 已排查信息

### 认证已通过（401 已解决）
- Client ID: `tuneloop_dev`
- Client Secret 已确认正确
- 已能成功获取 client credentials token（expires in 3600s）

### 权限不足（403）
```
# IAM 数据库中 tuneloop_dev client 信息
SELECT id, client_id, name, role FROM clients WHERE client_id = 'tuneloop_dev'
```

**推测根因**：`tuneloop_dev` client 缺少 `PermissionAdmin` 或同等权限角色，无法调用 `RegisterCustomerPermissions`。

## 请求协助
请确认：
1. `tuneloop_dev` client 在 IAM 中的角色是否包含权限注册能力？
2. 是否需要将 `tuneloop_dev` 的 role 提升为 `ADMIN` 或 `OWNER`？
3. 或者 `RegisterCustomerPermissions` endpoint 的权限检查是否需要调整？
