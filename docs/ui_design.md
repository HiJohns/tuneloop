# TuneLoop UI 设计文档

## 1. 概述

### 1.1 文档目的
本文档定义 TuneLoop 乐器租赁管理系统的界面设计规范，包括页面结构、功能点、交互逻辑和视觉设计原则。

### 1.2 技术栈
- **前端框架**: React 18 + Vite
- **UI 组件库**: Ant Design 5.x
- **路由**: React Router 6
- **状态管理**: React Hooks (useState, useContext)
- **样式**: Tailwind CSS + Ant Design 样式覆盖

### 1.3 页面入口
- **PC 管理后台**: http://localhost:5554
- **移动端**: http://localhost:5556

## 3. 页面设计

### 3.1 仪表盘 (Dashboard)

**路由**: `/dashboard`

**功能点**:
- 展示关键业务指标卡片
  - 乐器总数
  - 可租乐器数
  - 租赁中乐器数
  - 待处理维修工单数
- 租赁趋势图表
- 即将到期租赁列表
- 最近新增乐器

**交互逻辑**:
- 点击卡片可跳转至对应列表页面
- 图表支持时间范围筛选

### 3.2 乐器列表 (InstrumentList)

**路由**: `/instruments/list`

**功能点**:
- 乐器表格展示
  - 图片缩略图
  - 识别码 (SN)
  - 分类名称
  - 乐器等级
  - 网点
  - 状态标签
- 工具栏
  - 导入/导出
  - 搜索框
  - 分类筛选
  - 状态筛选
- 批量操作
  - 批量上架/下架
  - 批量修改价格
  - 批量删除
- 操作按钮
  - 编辑
  - 删除
  - 查看详情

**交互逻辑**:
- 点击行内编辑按钮 → 跳转 `/instruments/{id}/edit`
- 点击图片 → 跳转详情页 `/instruments/detail/{id}`
- 支持行选中进行批量操作
- 分页展示，每页默认 20 条

### 3.3 乐器表单 (InstrumentForm)

**重要变更** (2026-04-16):
- 乐器不再有 `name`（名称）字段，完全由 `sn`（识别码）标识
- 品牌、型号等属性作为动态属性存在

**路由**: 
- 新建: `/instruments/new/edit`
- 编辑: `/instruments/{id}/edit`

**功能点**:
- 基本信息
  - 识别码 (sn) - **唯一标识**
  - 分类 (category_id) - 树形选择
  - 等级 (level_id) - 下拉选择
  - 网点 (site_id) - 树形选择
- 动态属性
  - 根据系统配置的属性动态渲染
  - 例如：品牌、型号、颜色、年份等
  - 可在属性管理页面配置
  - 非必填
- 描述信息
- 图片上传
  - 支持拖拽排序
  - 上传进度显示
  - 失败重试
- 视频 URL

**表单验证**:
- 识别码: 必填，唯一性校验
- 分类: 必填
- 等级: 必填
- 网点: 必填
- 动态属性: 非必填

### 3.4 乐器详情 (InstrumentDetail)

## 4. 通用组件

### 4.1 受保护路由 (ProtectedRoute)

**功能**: 验证用户登录状态，未登录跳转至登录页

**逻辑**:
1. 检查 Token 存在性
2. 验证 Token 有效性 (检查过期时间)
3. Token 无效或过期 → 跳转 IAM 登录

### 4.2 图片上传组件 (SortableImageUpload)

**功能**: 支持拖拽排序的图片上传组件

**特性**:
- 拖拽调整顺序
- 上传进度显示
- 失败重试
- 图片预览

### 4.3 导入结果弹窗 (ImportResultModal)

**功能**: 展示导入结果

**展示内容**:
- 成功数量
- 失败数量
- 错误详情列表

### 4.4 树形选择组件 (TreeSelect)

**功能**: 支持动态加载子节点的选择组件

**用于**:
- 乐器分类选择
- 网点选择

### 4.5 删除确认 (DeleteConfirm)

**功能**: 操作前的二次确认

**特性**:
- Popconfirm 组件
- 危险操作使用红色按钮

## 5. 设计原则

### 5.1 布局规范
- 使用 24 栅格系统
- 内容区域 padding: 24px
- 卡片间距: 16px
- 表单项间距: 24px

### 5.2 颜色规范
- 主色: `#6366F1` (Indigo)
- 成功色: `#52C41A`
- 警告色: `#FAAD14`
- 危险色: `#F5222D`
- 信息色: `#1890FF`
- 文字色: `#262626` (主文字), `#8C8C8C` (次要文字)

### 5.3 状态颜色映射
```
乐器状态:
  - available (可租): green
  - rented (已租出): orange
  - maintenance (维修中): red

订单状态:
  - pending (待支付): orange
  - paid (已支付): blue
  - in_lease (租赁中): green
  - completed (已完成): gray
  - cancelled (已取消): gray
```

### 5.4 响应式设计
- PC 端: >= 1200px
- 平板端: 768px - 1199px
- 移动端: < 768px

### 5.5 交互反馈
- 按钮点击: 轻微缩放动画
- 加载状态: Spin 组件或骨架屏
- 操作成功: message.success
- 操作失败: message.error
- 表单错误: Form.Item validateStatus

### 5.6 表格规范
- 斑马纹: 不使用
- 边框: 带边框表格
- 悬停: 高亮当前行
- 选中: 蓝色边框高亮
- 分页: 右下角

### 5.7 表单规范
- 必填标识: 红色星号
- 标签对齐: 顶部对齐
- 错误提示: 输入框下方红色文字
- 占位符: 灰色提示文字

## 6. 登录流程

### 6.1 PC 端登录
1. 访问任意受保护页面
2. 未登录 → 跳转 IAM 授权页
3. 用户授权 → 回调 `/api/auth/callback`
4. 获取 Token → 保存至 localStorage
5. 跳转原请求页面

### 6.2 微信端登录
1. 访问移动端页面
2. 检测到微信浏览器 → 使用微信 OAuth
3. 授权成功 → 获取 Token
4. 保存 Token → 跳转首页

---

## 7. 错误处理

### 7.1 HTTP 401 处理
- 检测到 401 响应
- 清除本地 Token
- 跳转 IAM 登录页

### 7.2 HTTP 404 处理
- 返回统一的 404 页面
- 提供返回首页链接

### 7.3 网络错误处理
- 显示网络错误提示
- 提供重试按钮

### 7.4 表单验证
- 实时校验
- 提交时完整校验
- 错误信息定位到具体字段
