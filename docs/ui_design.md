# TuneLoop UI 设计文档

## 1. 概述

### 1.1 文档目的
本文档定义 TuneLoop 乐器租赁管理系统的界面设计规范，包括页面结构、功能点、交互逻辑和视觉设计原则。

### 1.2 技术栈
- **前端框架**: React 18 + Vite
- **UI 组件库**: Ant Design 5.x
- **路由**: React Router 6
- **状态管理**: React Hooks (useState, useContext)
- **样式**: Tailwind CSS + Ant Design 样式覆盖

### 1.3 页面入口
- **PC 管理后台**: http://localhost:5554
- **移动端**: http://localhost:5556

## 3. 页面设计

### 3.0 冷启动向导 (Setup)

**路由**: `/setup`  
**权限**: 无需登录（仅限未初始化系统访问）

**页面流程**:

1. **状态检测**: 页面加载时调用 `GET /api/setup/status`
   - 若 `requires_setup = false` → 自动重定向至登录页 `/`
   - 若 `requires_setup = true` → 显示初始化表单

2. **初始化表单**:
   - 邮箱（输入框，带格式验证）
   - 密码（密码框，强度提示）
   - 确认密码（密码框，一致性验证）
   - 『创建管理员』按钮（提交）

3. **提交处理**:
   - 表单验证通过后调用 `POST /api/setup/init`
   - 显示加载状态
   - 成功后后端返回 OIDC 授权 URL
   - 前端自动跳转至 IAM 完成首次认证

4. **错误处理**:
   - 系统已初始化（403）→ 显示错误并跳转登录页
   - 参数错误（400）→ 高亮显示错误字段

**交互细节**:
- 表单验证实时反馈
- 密码强度可视化（弱/中/强）
- 提交按钮禁用状态管理

### 3.1 商户管理 (MerchantManagement)

**路由**:
- 列表: `/merchants`
- 创建: `/merchants/new`
- 详情: `/merchants/:id`

**权限**: 仅 `project_admin` 角色可见

#### 3.1.1 商户列表页

**页面结构**:
- 顶部工具栏
  - 搜索框（按商户名称筛选）
  - 『创建商户』按钮（右上角）
- 表格区域
  - 列：商户名称、商户代码、联系人、管理员、创建时间、状态、操作
  - 行操作：查看详情、编辑、删除

**交互**:
- 分页加载（每页 20 条）
- 点击行进入详情页
- 删除操作弹出确认对话框，包含资产校验结果提示

#### 3.1.2 商户创建页

**表单字段**:
- 商户名称（必填）
- 商户代码（必填，格式验证，唯一性校验）
- 联系人信息：
  - 姓名
  - 邮箱（格式验证）
  - 电话（手机号格式验证）
- 指定管理员：
  - 点击『选择管理员』按钮
  - 弹出「指定用户对话框」（见 §3.2）
  - 选择后显示用户姓名和邮箱

**验证规则**:
- 商户代码：仅允许字母、数字、连字符
- 邮箱：标准邮箱格式
- 电话：11 位手机号

**提交**:
- 校验通过后调用 `POST /api/merchants`
- 成功后跳转至商户列表页

**错误处理**:
- 代码重复 → 提示"商户代码已存在"
- 管理员用户不存在 → 提示"请重新选择管理员"

### 3.2 指定用户对话框 (UserSelectionDialog)

**组件类型**: 可复用模态对话框

**使用场景**:
- 商户创建：选择管理员
- 网点管理：添加成员
- 人员管理：指定归属

**对话框结构**:

#### 3.2.1 顶部操作区

```
┌──────────────────────────────────────────────┐
│  选择用户                                   [×]│
├──────────────────────────────────────────────┤
│  [用户输入框]  [创建用户]                   │
└──────────────────────────────────────────────┘
```

**用户输入框** (`AutoComplete` 组件):
- 输入提示："输入用户名、邮箱或手机号搜索"
- 输入时 debounce 300ms 自动搜索
- 下拉框最多显示 10 条结果
- 每条结果显示格式：
  ```
  张三 (zhangsan@example.com)
  匹配: 邮箱    [已关联] ✓
  ```
  或
  ```
  李四 (13800000000)
  匹配: 手机号  [未关联] ⚠
  ```
- 点击结果添加到已选用户列表

**创建用户按钮**:
- 点击打开「创建用户子对话框」
- 即使搜索无结果也可直接创建

---

#### 3.2.2 已选用户列表

```
┌──────────────────────────────────────────────┐
│  已选择的用户 (2)                            │
├──────────┬──────────┬──────────┬──────────┤
│ 用户名   │ 邮箱     │ 手机号   │ 操作     │
├──────────┼──────────┼──────────┼──────────┤
│ 张三     │ zhang...@│ 138...   │ [删除]   │
│ 李四     │ lisi@... │ 139...   │ [删除]   │
│ ⚠ 该用户未与当前商户关联                   │
└──────────┴──────────┴──────────┴──────────┘
```

**列表列**:
- 用户名: 用户姓名
- 邮箱: 邮箱地址（过长时省略显示）
- 手机号: 手机号（过长时省略显示）
- 操作: 删除按钮

**未关联用户提示**:
- 若列表中存在 `associated=false` 的用户
- 在列表下方显示醒目提示（黄色背景）:
  ```
  ⚠ 以下用户尚未与当前商户关联，将通过邮件通知用户确认
  ```

**删除按钮**:
- 每行右侧的删除按钮
- 点击后从列表中移除该用户

---

#### 3.2.3 底部确认区

```
┌──────────────────────────────────────────────┐
│ ⚠ 如果用户列表中存在未关联用户，将通过 IAM     │
│    发送确认邮件，用户确认后完成关联。           │
│    网点成员添加为即时生效（下级组织仅通知）。   │
├──────────────────────────────────────────────┤
│                                   [取消] [确认]│
└──────────────────────────────────────────────┘
```

**未关联用户提示**（条件显示）:
- 仅当列表中存在 `associated=false` 的用户时显示
- 红色或黄色醒目标识
- 文本："商户管理员关联需用户在确认邮件中确认；网点成员添加即时生效（仅邮件通知）"

**按钮组**:
- 取消：关闭对话框，不保存选择
- 确认：返回已选用户列表，调用方继续提交

---

#### 3.2.4 创建用户子对话框

```
┌──────────────────────────────────────────────┐
│  创建新用户                                   [×]│
├──────────────────────────────────────────────┤
│  姓名:    [____________________________]     │
│  邮箱:    [____________________________]     │
│  手机号:  [____________________________]     │
│  密码:    [____________________________]     │
│                                              │
│                                   [取消] [创建]│
└──────────────────────────────────────────────┘
```

**表单字段**:
- **姓名**（必需）: 文本输入
- **邮箱**（必需）: 邮箱格式验证
- **手机号**（选填）: 手机号格式验证
- **密码**（必需）: 密码强度提示

**唯一性检查**:
- 实时检查：输入时自动验证 name/email/phone 是否已存在
- 检查 API: `GET /api/users/check?email=xxx&phone=xxx`
- **发现冲突时**:
  ```
  ┌──────────────────────────────────────────────┐
  │ ⚠ 发现重复用户信息                          │
  │                                              │
  │ 邮箱 zhangsan@example.com 已被以下用户占用:  │
  │ 张三 (ID: uuid)                              │
  │                                              │
  │ [选定该用户] [继续编辑]                      │
  └──────────────────────────────────────────────┘
  ```
  - **选定该用户**: 关闭创建对话框，将冲突用户添加到主对话框的已选列表
  - **继续编辑**: 标记冲突字段为错误状态，阻止表单提交

**创建后行为**:
- 新用户 status = pending
- 自动添加到主对话框的已选用户列表（associated=false）
- 关闭创建对话框

---

#### 3.2.5 组件 Props

```typescript
interface UserSelectionDialogProps {
  visible: boolean;
  onClose: () => void;
  onConfirm: (selectedUsers: SelectedUser[]) => void;
  merchantId: string;
  multiple?: boolean;  // 是否支持多选，默认 true
  title?: string;      // 对话框标题
}

interface SelectedUser {
  id: string;
  name: string;
  email: string;
  phone: string;
  associated: boolean;
}
```

**Props 说明**:
- `visible`: 对话框显示状态
- `onClose`: 关闭回调
- `onConfirm`: 确认回调，返回已选用户列表
- `merchantId`: 当前商户 ID（用于检查用户关联状态）
- `multiple`: 是否支持选择多个用户（默认 true）
- `title`: 自定义标题，默认"选择用户"

---

#### 3.2.6 交互细节

**键盘支持**:
- Enter：在 AutoComplete 中选中第一个结果
- Esc：关闭下拉框或关闭对话框
- Tab：在表单字段间切换

**加载状态**:
- 搜索时显示加载动画
- 创建用户时显示提交加载

**错误处理**:
- 搜索失败显示错误消息
- 创建用户失败显示具体错误
- 唯一性检查失败显示冲突信息

**性能优化**:
- AutoComplete 使用 debounce（300ms）
- 搜索结果本地缓存（5分钟）
- 已选用户列表使用虚拟滚动（如果超过50条）

### 3.3 乐器表单 (InstrumentForm)

**路由**: 
- 新建: `/instruments/new`
- 编辑: `/instruments/{id}/edit`

**功能点**:

1. **基本信息**
   - 识别码 (sn) - **唯一标识**，输入后自动校验唯一性
   - 分类 (category_id) - 树形选择，懒加载，提供链接跳转分类管理
   - 等级 (level_id) - 下拉选择（入门、专业、大师）
   - 归属网点 (site_id) - 树形选择
     - **租户管理员**：可选择任意网点
     - **网点管理员/成员**：默认锁定当前所属网点，不可修改

2. **动态属性**
   - 根据属性管理中定义的属性动态渲染输入控件
   - 支持下拉选择现有值或直接输入
   - **下拉框默认显示最常用的 3 个属性值**
   - 输入时实时搜索过滤，显示包含输入值的前 3 个常用项
   - 提供链接跳转到属性管理

3. **描述信息**
   - 文本输入框

4. **媒体上传**
   - 图片：最多 6 张，支持拖拽排序
   - 视频：最多 1 段（**支持文件上传**，非仅 URL）
   - 上传进度条显示
   - 失败自动重试
   - **流程**：先上传媒体文件 → 返回 UUID → 提交表单时附带 UUID

**表单验证**:
- 识别码: 必填，唯一性校验
- 分类: 必填
- 等级: 必填
- 网点: 必填（网点管理员/成员自动锁定）
- 动态属性: 非必填

**权限与行为**:
| 角色 | 网点选择 | 表单行为 |
|------|---------|---------|
| 租户管理员 | 可选任意网点 | 正常 |
| 网点管理员 | 锁定当前网点 | 不可修改 |
| 网点成员 | 锁定当前网点 | 不可修改 |

- 最后一名管理员：
  - 切换角色按钮禁用（灰色）

#### 3.10.2 删除网点增强

**删除前校验**:
- **资产检查**: 调用库存接口检查
  - 在库乐器（available）→ 弹窗提示: "该网点仍有 X 件在库乐器，请先转移资产"
  - 在租乐器（rented）→ 弹窗提示: "该网点仍有 X 件在租乐器，请先处理订单"
- **成员检查**: 若仍有成员 → 提示: "请先移除所有成员"

**删除流程**:
- 全部检查通过后弹出确认对话框
- 确认后调用 `DELETE /api/merchant/sites/:id`
- 成功后：左侧树移除节点，路由跳转至 `/sites`

### 3.16 维修会话 (MaintenanceSession)

**路由**: `/maintenance/sessions`

**权限**: 各角色不同视图
- 技师: 查看自己所属订单
- 员工: 查看网点所有订单

**功能点**:
- 维修订单列表
  - 日期、类别、简述、状态
  - 待分配状态显示『接单』按钮
  - 已分配状态显示在手订单标识
- 维修详情页
  - 问题描述和图片
  - 进度备注
  - 维修报告
  - 维修照片
- 技师操作（微信端）
  - 点击『接单』（待分配→已分配）
  - 点击『开始工作』（维修中状态）
  - 扫码乐器二维码
  - 输入评论、上传照片
  - 点击『完成』（验收状态）
- 员工验收（微信端）
  - 扫码乐器二维码
  - 按规定拍照上传
  - 点击『验收通过』（订单结束，在库状态）
  - 点击『验收不通过』（添加评论，待处理状态）

**状态流转**:
- 待分配 → 已分配（技师接单）
- 已分配 → 维修中（师傅开始工作）
- 维修中 → 验收中（师傅完工）
- 验收中 → 在库（员工验收通过）
- 验收中 → 待处理（员工验收不通过）

**交互逻辑**:
- 状态变更需记录操作人和时间
- 照片上传使用移动端相机组件
- 状态不可逆（除验收不通过回退到待处理）
- 通知机制：状态变更推送消息给相关人员

### 3.17 申诉处理 (AppealManagement)

  - 完整信息展示

- 申诉提交后通知经理
- 仲裁决策需填写完整说明
- 调整金额需合理范围验证
