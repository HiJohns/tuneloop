# 乐器租赁系统功能需求规格说明书 (v26.3.16)

## 一、 用户端（微信小程序）

用户端面向最终租机用户，重点在于商品展示、便捷租赁及维保服务闭环。

### 1. 注册与登录
* **微信快捷登录**：支持微信授权获取手机号一键登录/注册。
* **状态保持**：支持 30 天免登录，提升用户回访体验。

### 2. 乐器租赁中心
* **分类与列表**：支持一级（乐器品类）与二级（品牌/型号）分类；支持按租金、品牌、库存状态等条件筛选排序。
* **详情展示**：多图轮播及视频介绍；详细展示参数（材质、尺寸、适用人群等）。
* **计费联动**：
    * **规格选择**：不同规格对应不同租金/押金。
    * **阶梯定价**：支持入门/专业/大师级分级定价。
    * **租期折扣**：支持 3/6/12 个月租期选择，12 个月自动触发 95 折。
* **租转售 (Rent-to-Own)**：明确标注“租满 12 个月归个人所有”的核心逻辑。

### 3. 订单与支付
* **免押金体系**：支持会员等级或信用分免押金模式（模拟/接入）。
* **首期汇总**：清晰展示“首月租金 + 押金”的总额。
* **协议签署**：集成在线租用协议，确保租赁合规。

### 4. 维保服务门户
* **在线报修**：支持上传故障图片、描述问题；支持自送网点或上门取件选择。
* **维保记录**：实时追踪工单进度（已报修、维修中、已修复）。报修单创建自动生成创建记录；维修过程可追加带照片/视频的进度记录，视频保留最新一条。

### 5. 个人中心
* **租约管理**：可视化展示当前租约、已租时长及**租转售进度条**。
* **资产所有权**：租满 12 个月后，支持查看/下载“电子所有权转移证明”。
* **我的收藏/地址**：管理意向商品及配送信息。

### 6. 购物车与批量下单 *
* **购物车管理**：支持将多个可租乐器加入购物车，按租户分组显示。
* **批量下单**：购物车中按租户分组，支持一键下单接入现有流程（登录→确认收货地址、支付押金）。
* **状态管理**：购物车数据存储于本地（localStorage），下单后自动清空。

### 7. 公共浏览（游客模式） *
* **游客访问**：无需登录即可浏览乐器列表和详情。
* **租户筛选**：支持 URL 参数 `?tenant=` 过滤指定商户的乐器。

## 二、 商家管理端（PC端）

商家端面向具体的直营店或加盟网点，侧重于资产运营与订单处理。

### 1. 资产与库存管理
* **设备台账**：SN 码唯一标识管理，记录乐器分级、型号及折旧情况。
* **库存监控**：实时查看网点在库/在租/维保状态。
* **所有权监控**：标记即将转售（租期满 12 个月）的资产。

### 2. 租赁业务管理
* **租约台账**：查看所有在租订单、到期时间、押金流水。
* **逾期预警**：自动标记逾期未归还资产，支持一键发送提醒。

### 3. 维保调度管理
* **工单列表**：展示网点所有报修单；支持接单、分配师傅、确认取琴。
* **进度更新**：上传维修过程描述，同步至用户小程序端。
* **报价中心**：针对超出服务包范围的维修项进行线上报价。

### 4. 财务结算
* **佣金明细**：查看本网点租赁/维保业务的佣金结算状态。
* **流水报表**：支持按时间段导出业务收入明细。

### 5. 组织管理
* **网点管理**：树状展示网点列表，懒加载模式，支持创建顶级网点、查看详情、编辑、删除。
* **人员管理**：人员列表展示，支持按姓名和网点搜索，负责人可点击跳转至人员详情。

### 6. 用户与权限
* **用户创建**：创建用户时校验邮箱/手机号唯一性，冲突时弹出选择对话框。支持指定角色（site_member / site_admin / worker），未指定时首成员默认 site_admin，后续默认 site_member。
* **用户编辑**：编辑用户信息，归属网点下拉框采用树状展示。
* **冲突处理**：唯一性冲突时提供选择对话框，支持"继续创建"或"选择已有用户"。
* **状态同步**：用户页面"从 IAM 同步"按钮同步 name、email、phone、role、status 到本地；新建用户默认 status=pending 与 IAM 一致。

## 三、 平台运营端（总控端）

平台端面向系统超级管理员，侧重于全局治理、租户管理及财务结算。

### 1. 商家与权限管理
* **商家准入**：审核加盟商家资料，收取入驻押金。
* **商户分类**：支持两种商户类型 —— **全权商户**（full）与 **受控商户**（controlled）。受控商户在商户层面设立中转地址（含电话），用户下单和归还时均使用中转地址，完全隔离消费者与商户之间的直接联系。
* **RBAC 权限**：配置各级组织管理员、网点管理员及维保人员权限。

### 2. 全局计费规则
* **定价矩阵**：配置不同级别乐器的租金系数、押金比例、续租折扣。
* **阶梯房租**：默认定价策略 —— 前 30 天基准价、31–180 天 95 折、180 天后 9 折；逾期按基准价 ×2 计费。
* **押金规则**：押金按乐器总价 × 押金比例（默认 30%）计算。网点管理员可手动覆盖押金金额。
* **维保服务包**：定义各级别乐器包含的免费维保内容（如大师级含专家精调）。

### 3. 财务与结算中心
* **全局结算**：设置平台佣金比例（周结/月结）。
* **押金监管**：监管全平台押金流水，处理押金扣除或原路退还审核。

### 4. 资产全生命周期审计
* **流转轨迹**：查看单台资产从入库到转售的全路径轨迹。
* **统计大屏**：全局在租率、资产总估值、转售转化率等核心指标监控。

### 5. 媒体管理
* **图片/视频上传**：支持 JPEG/PNG/GIF/WebP/MP4/WebM/MOV 格式上传。
* **批次管理**：按 shipping/forwarding/accepting/returning/relaying/receiving/repaired 类型分组管理媒体。
* **展示设置**：指定某批次为乐器展示媒体，多图轮播 + 视频介绍。
* **视频缩略图**：上传视频后自动提取第 1 秒帧作为缩略图（需 FFmpeg）。
* **存储模式**：支持本地存储和 OSS 双模式，通过 `OSS_ENDPOINT`/`OSS_BUCKET` 环境变量切换。
* **上传大小限制**：namespace_admin 可配置图片（默认 10MB）/视频（默认 100MB）上限。
* **定时清理**：MediaCleanupService 扫描超期媒体（保留 5 年），输出待清理列表供管理员确认。

### 6. 审计日志
* **操作溯源**：记录所有 CRITICAL 和 HIGH 级别操作的审计日志（用户/商户/订单/站点/权限等变更）。
* **查询检索**：支持按时间范围、操作类型、资源类型、用户筛选。
* **RBAC 可见范围**：普通用户仅本人日志，网点管理员见本组织，租户管理员见全租户。
* **导出能力**：支持 CSV 格式导出审计日志。

## 四、 项目开发说明（参考）

* **架构模式**：采用前端分端口（小程序/PC端）+ 后端统一 API 架构。
* **身份底座**：集成 **Lin-IAM** 实现多租户隔离与白标化（White-labeling）适配。
* **核心逻辑**：
    * **租转售状态机**：由系统后台定时器自动执行。
    * **12个月计费计算器**：前端实时计算，后端签名校验防止篡改。

## 五、 会员与促销系统

### 5.1 会员级别

系统支持三级会员（初级/中级/高级），按跨商户累计消费金额自动升级，仅升级不降级。会员级别影响乐器租赁价格计算（会员折扣）。

### 5.2 会员折扣政策

由系统管理员（sys_admin）设置系统级默认折扣方案，商户管理员（merchant_admin）可采纳系统方案或创建本商户方案覆盖。折扣政策按会员级别定义不同的租金/押金折扣率。

### 5.3 返点政策

系统管理员按会员级别设置不同的租金→点数返还比例。商户管理员仅可决定本商户是否参与返点。网点管理员可逐乐器决定是否适用返点。

### 5.4 乐器级促销覆盖

网点管理员可为每件乐器独立开启/关闭折扣政策和返点政策的适用性（不影响政策本身）。

### 5.5 点数政策

促销点数政策三级可覆盖（网点＞商户＞系统），定义点数可支付上限和有效期。

## 六、 租金计算系统

### 6.1 计算引擎

租金计算由 `backend/services/rent_calculator.go` 统一处理，核心公式：

```
实际租金 = 基础日租金 × 阶梯折扣 × (促销覆盖 ? 会员折扣 : 1.0) × 促销折扣 × 租期天数
```

**折扣要素**：
| 折扣类型 | 来源 | 说明 |
|---------|------|------|
| 阶梯折扣 | 租期长度 | ≤30 天: 1.0, 31–90 天: 0.95, 91–180 天: 0.90, >180 天: 0.85 |
| 会员折扣 | membership_levels | 初级: 1.0, 中级: 0.95, 高级: 0.90 |
| 促销折扣 | promo_plan_details | 按 instrument_promo_overrides 判断是否适用 |
| 逾期费率 | promo_plan_details (overdue_discount) | 默认 1.5× 日租金 |

### 6.2 点数钱包

`backend/handlers/user_points.go` 提供三点管理：
- **预付点**（prepaid_points）— 用户充值购买，可用于租金抵扣
- **赠点**（promo_points）— 系统赠送/返点，仅按 points_policy.cap_rate 比例抵扣
- 还款时优先扣预付点，赠点受上限限制

### 6.3 订单支付点数抵扣

`POST /api/user/orders`（CreateOrder）在创建订单时：
1. 接收 `prepaid_points_used`、`gift_points_used` 参数
2. 校验预付点余额 ≥ 提议使用量
3. 校验赠点使用量 ≤ `floor(总实付 × cap_rate / 100)`
4. 扣减对应余额（prepaid_points / promo_points）
5. 记录 `PointsTransaction`（type=prepaid_deduction / gift_deduction）
6. 保存 `pricing_breakdown` 和 `points_policy_snapshot` JSON 快照

### 6.4 逾期自动扣款

定时任务（每日凌晨 0:05）扫描 `status=in_lease AND end_date < today` 订单：
1. 逾期租金 = `monthly_rent / 30 × overdue_discount_rate`
2. 优先从 prepaid_points 扣款，不足部分记 `remaining_balance`
3. 状态：success / partial / failed
4. 失败/部分 → 通知用户 + 网点经理

### 6.5 归还结算

用户归还乐器时触发结算（`backend/handlers/user_settlement.go`）：
1. 按实际租期重算租金（使用 pricing_breakdown 中的 final_daily_rent）
2. 赠点调整：`gift_used > floor(actual_rent × cap_rate / 100)` → 退回差额
3. 退款拆解：可提现 ≤ 原实付现金，余款退回预付点
4. 逾期扣款计入 settlement.overdue_charges_total
5. 确认后更新 order.status = completed，调整用户余额

### 6.6 首次登录引导

OAuth 登录后未完成引导的用户跳转 `/onboarding`，可选：
- 设置昵称
- 添加收货地址（通过 `POST /api/user/addresses`）
- 上传身份证照片
- 预购点数（调用 `POST /api/user/points/purchase`）
