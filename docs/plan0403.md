# TuneLoop 系统重构与功能演进计划书 (2026-04)

## 一、 背景与核心架构调整
将系统逻辑从“工业化标品”转向“高价值非标资产”管理模式。核心原则：**一物一码、资产中心化、流程闭环化。**

## 二、 核心修改点总结

### 1. 基础设施与鉴权 (Infrastructure & Auth)
* **统一 API 调用**：封装 `api.get/post`，集成 JWT 自动注入与 401 全局拦截。
* **登录态优化**：实现“滑动窗口”静默续期，支持 30 天长效登录；适配微信 Webview 环境下的 401 跳转。
* **UUID 严格校验**：后端及数据库统一处理空值（`NULL` 而非 `""`），防止 PG 类型报错。

### 2. 数据模型重构 (Data Schema)
* **非标属性支持**：使用 PostgreSQL `JSONB` 存储 `metadata`（制作师、产地、材质等）和 `specifications`。
* **媒体矩阵**：支持多图上传及第三方视频链接（B站、优酷）解析与嵌套展示。
* **标签归一化 (Label Normalization)**：支持“自由录入+后台审计”模式，建立标准标签与别名映射表，解决标签漂移。

### 3. UI/UX 体验升级
* **列表页扁平化**：移除“查看”按钮，名称与图像转为链接；实现“父子行”可展开结构展示多规格信息。
* **编辑模式转换**：弃用 Modal 对话框，改为全屏编辑页；支持规格与价格（x6/x25）的自动联动计算。
* **视觉优化**：全局应用方案三（悬浮式自动隐藏）滚动条；重构仪表盘为“控制塔”模式，移除冗余列表。

### 4. 租赁安全闭环 (Security & Workflow)
* **出库确认（数字握手）**：用户在小程序端预览“入库照片”并勾选确认后，资产状态方可变更。
* **归还审计（证据链）**：员工按模板拍照，系统提供“出库 vs 归还”双栏同屏比对 UI 进行定损。
* **法律确权**：集成 HTML5 Canvas 电子签名；后端自动生成 PDF 格式的《归还鉴定报告》作为终审凭证。

## 四、 补充建议 (CTO Perspectives)

1.  **资产 ID (QR Code)**：为了配合员工在现场“按规定流程拍照”，建议在“乐器入库”后自动生成一个**唯一的 QR 码**。员工在归还时“扫码即拍”，能极大减少人工选择乐器的错误率。
2.  **网点库存看板**：既然库存是分布在各网点的，建议在“库存监控”页增加一个**“地图/分布试图”**，一眼看出某型号乐器在多伦多各店的调拨余量。
3.  **缓存策略**：由于“自下而上”的搜索涉及大量 JSONB 聚合，当资产超过 1000 件时，建议对仪表盘的统计数字引入 **Redis 缓存**，每 5 分钟更新一次。
