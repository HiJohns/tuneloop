## 🎯 阶段1：乐器展示闭环（优先实现）

### 1.1 [Mobile-5553] 微信小程序端

#### M-001: 用户登录与授权【基础】
- **所属模块**: 注册/登录
- **任务描述**: 
  - 实现微信授权快速获取手机号登录（使用微信小程序手机号组件）
  - 登录状态保持：JWT token本地存储，30天免登录，过期自动退出
  - 首次登录自动完成注册，自动创建用户档案（头像、昵称、open_id、union_id）
- **API依赖**: `[Backend-API]-A-001` 用户认证API
- **涉及页面**: `pages/auth/login`

#### M-002: 乐器分类页【核心】
- **所属模块**: 乐器展示
- **任务描述**:
  - 展示一级分类（钢琴、吉他、古筝、小提琴等）
  - 二级分类筛选（按品牌、按类型）
  - 分类支持自定义排序，可隐藏/显示指定分类（后端控制）
  - 分类图标展示（从后端动态加载）
- **API依赖**: `[Backend-API]-C-001` 分类列表API
- **涉及页面**: `pages/category/index`

#### M-003: 乐器列表页【核心】
- **所属模块**: 乐器展示
- **任务描述**:
  - 展示核心信息：乐器名称、首图、租金（日/月）、押金、实时库存状态
  - 筛选功能：按租金区间、品牌、租期类型、库存状态（有货/无货）
  - 排序功能：按租金从低到高/从高到低、按上新时间
  - 无限滚动分页加载
- **API依赖**: `[Backend-API]-C-002` 乐器列表API, `[Backend-API]-C-003` 库存查询API
- **涉及页面**: `pages/instrument/list`

#### M-004: 乐器详情页【核心】
- **所属模块**: 乐器展示
- **任务描述**:
  - 展示基础信息：名称、品牌、型号、材质、尺寸、适用人群
  - 多媒体：多图轮播（支持放大查看）、视频介绍（可选）
  - 规格选择：不同规格对应不同租金/押金（阶梯定价）
  - 明确标注日租/周租/月租单价（周租=日租×6，月租=日租×25等优惠规则）
  - 实时库存显示：实时显示可租赁数量，无货时标注"暂无可租"
  - 配送服务选择：门店自提/线下闪送
  - 用户评价展示：评分、文字、图片
- **API依赖**: `[Backend-API]-C-004` 乐器详情API, `[Backend-API]-C-005` 评价列表API
- **涉及页面**: `pages/instrument/detail`

### 1.3 [Backend-API] 后端服务

#### B-001: 用户认证API【基础】
- **接口路径**: POST /api/auth/wx-login
- **功能描述**:
  - 接收微信小程序code，调用微信API获取open_id/union_id
  - 查询用户表，不存在则自动注册（创建用户档案）
  - 生成JWT token（含user_id, role），有效期30天
  - 返回token及用户信息

#### B-002: 分类管理API族【基础】
- **接口列表**:
  - GET /api/categories - 分类列表（小程序和管理后台共用）
  - POST /api/categories - 创建分类（管理后台）
  - PUT /api/categories/:id - 更新分类（管理后台）
  - DELETE /api/categories/:id - 删除分类（管理后台，需检查是否被引用）
- **功能描述**: 分类CRUD，支持多级分类、排序、显示状态控制

#### B-003: 乐器管理API族【核心】
- **接口列表**:
  - GET /api/instruments - 乐器列表（支持筛选、排序、分页）
  - GET /api/instruments/:id - 乐器详情
  - POST /api/instruments - 创建乐器（管理后台）
  - PUT /api/instruments/:id - 更新乐器（管理后台）
  - PUT /api/instruments/:id/shelf-status - 上下架控制
- **功能描述**: 乐器CRUD，支持多规格管理、价格策略、媒体文件管理

#### B-004: 库存管理API族【核心】
- **接口列表**:
  - GET /api/instruments/:id/stock - 实时库存查询
  - PUT /api/instruments/:id/stock - 库存调整（管理后台）
  - GET /api/stock/transaction-log - 库存变动日志
- **功能描述**: 库存实时查询、库存调整、操作日志记录

#### B-005: 评价管理API族【辅助】
- **接口列表**:
  - GET /api/instruments/:id/reviews - 评价列表
- **功能描述**: 读取评价数据（评论+评分+图片）

#### B-006: 文件上传API【通用】
- **接口路径**: POST /api/upload
- **功能描述**: 支持图片、视频上传，返回URL，集成OSS或本地存储

## 🔄 阶段2：租赁闭环 (Lease-to-Return Loop)

> 从用户下单 → 支付 → 商家接单 → 发货 → 用户归还 → 押金退还的完整租赁生命周期

### 2.1 用户端流程 (Mini Program)

#### M-005: 租赁下单【核心】
- **所属模块**: 租赁功能
- **任务描述**:
  - 在乐器详情页选择规格、租期（日租/周租/月租）、数量
  - 选择配送方式（上门配送或到店自取）
  - 选择地址（配送方式）或门店（自取方式）
  - 系统自动计算租金、押金、配送费
  - 展示费用明细：租金、押金、配送费、总金额
  - 确认无误后提交订单
- **涉及页面**: `pages/rental/order-confirm`
- **API依赖**: `[Backend-API]-R-001` 创建租赁订单API

#### M-006: 订单支付【核心】
- **所属模块**: 租赁功能
- **任务描述**:
  - 调起微信支付，支付租金+押金（全额）
  - 支付超时15分钟自动取消订单
  - 支付成功后将订单状态更新为"待取琴"
  - 支持重新支付（针对支付失败或超时未支付）
- **涉及页面**: `pages/rental/payment`
- **API依赖**: `[Backend-API]-R-002` 支付API, `[Backend-API]-R-003` 订单状态查询API

#### M-007: 订单列表与查看【核心】
- **所属模块**: 订单中心
- **任务描述**:
  - 展示所有租赁订单
  - 分类筛选：全部、待支付、待取琴、租赁中、待归还、已完成、已取消
  - 支持按时间（近7天/近30天/全部）、订单号搜索
  - 展示核心信息：订单号、乐器图片名称、订单状态、租期、金额
- **涉及页面**: `pages/order/rental-list`
- **API依赖**: `[Backend-API]-R-004` 租赁订单列表API

#### M-008: 订单详情与操作【核心】
- **所属模块**: 订单中心
- **任务描述**:
  - 展示订单详细信息：乐器信息、租期、租金、押金、配送方式、地址
  - 展示支付信息：支付方式、支付时间、交易单号
  - 展示取琴/归还信息（时间、地点、状态）
  - 状态特定操作按钮：
    - 待支付：去支付、取消订单
    - 租赁中：续租申请、归还申请
    - 已完成：查看退还押金进度
- **涉及页面**: `pages/order/rental-detail`
- **API依赖**: `[Backend-API]-R-005` 订单详情API, `[Backend-API]-R-010` 取消订单API

#### M-009: 续租申请【核心】
- **所属模块**: 租赁功能
- **任务描述**:
  - 在"我的订单"中发起续租申请（仅限租赁中订单）
  - 选择续租时长，系统自动计算续租租金
  - 展示续租明细（无需重复支付押金）
  - 支付续租费用
  - 续租后订单到期时间同步更新
  - 续租申请需管理员审核（或自动通过）
- **涉及页面**: `pages/order/rental-renew`
- **API依赖**: `[Backend-API]-R-006` 续租申请API, `[Backend-API]-R-007` 续租支付API

#### M-010: 归还申请【核心】
- **所属模块**: 租赁功能
- **任务描述**:
  - 发起归还申请（支持未到期或已到期归还）
  - 选择归还方式：上门取琴（标注取琴费）或到店归还
  - 填写归还备注（乐器状态、使用感受）
  - 提交后等待管理员确认
- **涉及页面**: `pages/order/rental-return-apply`
- **API依赖**: `[Backend-API]-R-008` 归还申请API

#### M-011: 押金退还查看【核心】
- **所属模块**: 个人中心-押金管理
- **任务描述**:
  - 在"押金管理"中查看押金明细
  - 展示未退还/已退还的押金列表
  - 展示对应订单号、金额、退还时间
  - 租赁完成且无损坏时，显示"可申请退还"按钮
  - 展示押金退还申请审核进度
- **涉及页面**: `pages/user/deposit`
- **API依赖**: `[Backend-API]-R-009` 押金查询API, `[Backend-API]-R-012` 押金退还申请API

### 2.2 管理端流程 (Admin Backend)

#### A-007: 租赁订单管理列表【核心】
- **所属模块**: 租赁订单管理
- **任务描述**:
  - 展示所有租赁订单（平台视角）或商家自身订单（商家视角）
  - 筛选条件：订单号、用户手机号、订单状态、下单时间、乐器类型
  - 展示核心字段：订单号、用户信息、乐器信息、租期、金额、支付状态、配送方式
  - 批量操作：批量导出、批量修改状态、批量取消订单
- **涉及页面**: `pages/order/rental-manage`
- **API依赖**: `[Backend-API]-R-004` 租赁订单列表API

#### A-008: 订单状态流转操作【核心】
- **所属模块**: 租赁订单管理
- **任务描述**:
  - **待支付状态**：手动取消订单、发送支付提醒（短信/小程序消息）
  - **待取琴状态**：确认取琴（记录取琴时间、取琴人）、修改配送/自取信息
  - **租赁中状态**：处理续租申请（审核通过/拒绝）、处理归还申请、设置逾期计费（手动/自动）
  - **待归还状态**：确认归还（记录归还时间、乐器状态、是否损坏）、扣除滞纳金（如有）
  - **已完成状态**：审核押金退还申请，发起退款（微信支付原路退回）
  - **通用操作**：添加订单备注，同步至小程序订单详情
- **审批流程设置**：续租申请可配置为自动通过或需人工审核
- **涉及页面**: `pages/order/rental-detail-manage`
- **API依赖**: `[Backend-API]-R-013` 订单状态更新API, `[Backend-API]-R-014` 续租审核API, `[Backend-API]-R-015` 归还确认API

#### A-009: 逾期订单管理【核心】
- **所属模块**: 租赁订单管理
- **任务描述**:
  - 逾期订单列表：展示逾期订单、逾期天数、滞纳金金额
  - 滞纳金自动计算：按配置规则（如日租1.5倍）自动计算，支持手动调整
  - 催收提醒：手动发送短信/小程序消息提醒用户归还
  - 支持一键生成逾期费用报表
- **涉及页面**: `pages/order/overdue-manage`
- **API依赖**: `[Backend-API]-R-016` 逾期订单查询API, `[Backend-API]-R-017` 滞纳金计算API

#### A-010: 押金管理【核心】
- **所属模块**: 财务与结算管理
- **任务描述**:
  - 押金列表：展示未退还/已退还的押金，关联订单、用户、金额
  - 押金退还审核：审核用户押金退还申请（检查订单状态、乐器状态、逾期费用）
  - 发起退款：审核通过后对接微信支付退款接口，记录退款金额、时间、状态、交易单号
  - 支持批量退款（针对多个订单统一退还押金）
  - 退款记录查询与导出
- **审批流程**：可配置多级审核（如超过一定金额需财务主管审批）
- **涉及页面**: `pages/finance/deposit-manage`
- **API依赖**: `[Backend-API]-R-018` 押金审核API, `[Backend-API]-R-019` 微信支付退款API

### 2.3 租赁闭环Backend API

#### R-001: 创建租赁订单API【核心】
- **接口路径**: POST /api/rental-orders
- **功能描述**:
  - 接收订单参数：用户ID、乐器ID、规格ID、租期类型、租期长度、数量、配送方式、地址ID
  - 计算订单金额：根据租期长度和阶梯定价规则计算租金，根据押金规则计算押金
  - 生成唯一订单号（可支持按日期生成，如R2026032400001）
  - 状态初始化：创建后为"待支付"
  - 库存预占用（可选，避免超卖）
  - 返回订单详情及支付参数

#### R-002: 租赁订单支付API【核心】
- **接口路径**: POST /api/rental-orders/:orderId/pay
- **功能描述**:
  - 调用微信支付统一下单接口
  - 接收支付结果回调，更新订单状态为"待取琴"
  - 支付超时15分钟未支付则自动取消订单，释放库存

#### R-003: 租赁订单查询API【核心】
- **接口路径**: GET /api/rental-orders
- **功能描述**:
  - 查询当前用户的租赁订单列表（小程序端）
  - 支持按状态、时间筛选
  - 分页返回（每页10条）
  - 返回字段：订单号、乐器信息、租期、金额、状态、下单时间

- **接口路径**: GET /api/rental-orders/:orderId
- **功能描述**: 查询单个订单详细信息

- **接口路径**: GET /api/rental-orders/all（管理端）
- **功能描述**: 查询所有租赁订单（平台/商家视角），支持多维筛选

#### R-004: 续租API【核心】
- **接口路径**: POST /api/rental-orders/:orderId/renew
- **功能描述**:
  - 接收续租时长参数
  - 根据当前订单到期时间和续租时长计算新到期时间
  - 计算续租租金（根据阶梯定价，无需押金）
  - 生成续租记录，状态为"待支付"
  - 支付后更新主订单的到期时间

#### R-005: 归还申请API【核心】
- **接口路径**: POST /api/rental-orders/:orderId/return
- **功能描述**:
  - 接收归还方式、归还时间、备注信息
  - 更新订单状态为"待归还"（商家确认前）
  - 生成归还申请记录
  - 推送消息通知给商家管理员

- **接口路径**: POST /api/rental-orders/:orderId/return/confirm（管理端）
- **功能描述**: 商家确认归还，更新状态为"已完成"，触发押金退还流程

#### R-006: 订单状态流转API【核心】
- **接口路径**: PUT /api/rental-orders/:orderId/status
- **功能描述**: 
  - 根据不同操作更新订单状态：待支付→待取琴→租赁中→待归还→已完成
  - 每个状态流转需记录操作人、操作时间、备注

#### R-007: 滞纳金计算API【核心】
- **接口路径**: GET /api/rental-orders/:orderId/late-fee
- **功能描述**:
  - 计算逾期天数（基于当前时间和订单到期时间）
  - 根据配置规则（如1.5倍日租）计算滞纳金金额
  - 支持手动调整滞纳金（管理员可修改）
  - 记录滞纳金计算日志

#### R-008: 押金管理API【核心】
- **接口路径**: POST /api/deposit/refund-request
- **功能描述**: 
  - 接收用户押金退还申请
  - 校验订单状态（必须已完成且无逾期/损坏）
  - 生成押金退还记录，状态为"待审核"

- **接口路径**: POST /api/deposit/refund-requests/:requestId/review（管理端）
- **功能描述**: 管理员审核押金退还，通过则调用微信退款接口

#### R-009: 订单统计API【辅助】
- **接口路径**: GET /api/rental-orders/stats
- **功能描述**:
  - 统计租赁订单数据：今日/昨日/本月订单数、交易额、押金总额
  - 按乐器类型、租期、时间段统计
  - 按订单状态分布统计

## 🔧 阶段3：维保闭环 (Maintenance-to-Finish Loop)

> 从用户报修 → 商家接单 → 报价 → 用户支付 → 维修完成 → 乐器归还的完整维保生命周期

### 3.1 用户端流程 (Mini Program)

#### M-012: 维修/保养套餐展示【核心】
- **所属模块**: 乐器维护/维修功能

#### M-013: 维修/保养预约下单【核心】
- **所属模块**: 乐器维护/维修功能
- **任务描述**:
  - 选择服务类型：保养 / 维修
  - 选择乐器类型、具体型号（可选，从订单历史中选择）
  - 填写故障描述：文字描述 + 上传图片/视频（最多5张图 / 1个视频）

#### M-014: 维修进度查看【核心】

- **API依赖**: `[Backend-API]-M-003` 维修订单详情API, `[Backend-API]-M-004` 进度查询API

#### M-015: 维修订单列表【核心】
- **所属模块**: 订单中心
- **任务描述**:
  - 展示所有维修订单
  - 分类筛选：全部、待接单、待取琴、维修中、待报价、待支付、已完成、已取消

  - 展示字段：订单号、用户信息、乐器信息、服务类型、故障描述、报价金额、预约时间

- **任务描述**:

  - 报价区间校验：需在平台设定的合理区间内（避免恶意高价）
  - 查看用户报价确认状态（待确认/已确认/已拒绝）

- **API依赖**: `[Backend-API]-M-010` 报价提交API

### 4.1 商家入驻与审核模块 (Merchant Onboarding)

#### A-016: 商家入驻申请【核心】

## 📊 补充说明

### "纵向闭环"思维的核心价值

相比传统的"横向功能模块开发"（先开发所有小程序功能，再开发所有后台功能），**纵向闭环**强调：

1. **端到端交付**: 每个阶段都形成一个可独立运行的完整业务流程（如租赁闭环从下单到归还）
2. **快速验证**: 优先实现MVP闭环，让团队和用户可以快速验证核心价值的可行性
3. **降低风险**: 避免"开发了一堆功能但无法形成闭环"的风险
4. **增量扩展**: 每个闭环都是前一个闭环的增量扩展（展示 → 租赁 → 维保 → 平台）

### 任务拆分原则

- **原子性**: 每个任务开发周期控制在2-4小时内，可在一个工作日内完成
- **独立性**: 任务之间尽可能独立，减少依赖阻塞
- **可测试**: 每个任务完成后都有明确的验收标准
- **可追溯**: 所有任务都关联到具体的业务需求和用户价值
